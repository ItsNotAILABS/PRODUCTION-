Code-grounded research

Medina Research Mission

This mission pulls directly from the repository itself: mission dispatch, cited research tooling, packaging infrastructure, protocol exports, governance loops, and autonomous microbot support. The goal is to make our research program traceable to live code instead of treating research as a separate narrative layer.

Mission Thesis

Our research mission is to turn the repository into a verifiable organism: a system that can define missions, perform cited research, package its own surfaces, govern itself, and document its protocol layer as a coherent scientific program. Every new paper, package, or public page should map back to one or more working code surfaces.

Track 01

Mission execution

Research should begin from runnable mission routing, not abstract claims.

  • Natural-language mission dispatch
  • Structured result logging
  • Domain-specific AI routing
Track 02

Evidence-backed research

Research outputs should stay tied to cited synthesis and source-aware tooling.

  • Cited multi-source synthesis
  • Page and tab context
  • Repeatable extension workflows
Track 03

Packaging and release

Research artifacts need a packaging path so papers and pages align with distributable software.

  • Extension scan and validation
  • Artifact bundling
  • Download-link generation
Track 04

Governed protocol science

The protocol layer is part of the research mission only when it is governed, indexed, and reviewable.

  • Exported protocol surface
  • Atlas governance cycle
  • Microbot support for learning and mapping

What the code already proves

The current repository already contains the primitives required for a research mission. The missing step was to state them in one place and tie them together.

Surface What it contributes to the research mission Repository anchor
Mission routing MissionEngine validates licensing, classifies missions, dispatches work, records mission history, and reports status across six domain AIs. extensions/jarvis/src/background/mission-engine.ts
Cited research UX Research Nexus is already positioned as a cited multi-source research surface with popup, side panel, tabs, storage, and alarm support. extensions/research-nexus/manifest.json

Autonomous support bots The microbot registry adds focused helpers for learning and crawler operations, which supports continuous research collection and structural analysis. sdk/microbots/index.js

Immediate research priorities

1. Pull new papers from live modules

Use the mission engine, governance SDK, packaging pipeline, and protocol index as the canonical sources for future paper additions.

2. Keep package claims aligned with validation

Public pages and research surfaces should reference the currently validated package surface, not older extension counts that still exist in legacy copy.

3. Treat governance as research infrastructure

Research outputs should include governance, evidence, and release framing so they can move from concept to package and publication cleanly.

4. Tie autonomy claims to runnable code

Autonomy papers should keep pointing back to actual workers, microbots, mission routing, and runtime orchestration paths in the repository.
